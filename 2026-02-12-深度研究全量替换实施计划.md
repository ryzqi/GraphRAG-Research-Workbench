# 深度研究全量替换（DeepAgents 单引擎）Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将现有“深度研究”链路彻底替换为 deepagents 单引擎，实现全流式、可中断恢复、双产物（Markdown+JSON）输出，前后端与数据层全部重构。  

**Architecture:** 后端以 `create_deep_agent` 为唯一执行入口，保留 deepagents 默认中间件能力（TodoList/Filesystem/SubAgent/Summarization），并启用 `skills + memory + interrupt_on + checkpointer + store`。研究过程采用“会话-事件-工件”模型持久化，前端改为事件驱动研究工作台。研究模块不再接入 MCP，外部检索统一走内置工具链。  

**Tech Stack:** Python 3.13、FastAPI、deepagents、LangGraph（checkpoint/store）、LangChain、Celery、SQLAlchemy、PostgreSQL、Next.js + React + TypeScript、SSE

## 0. 执行原则（全任务适用）

- **技能约束：** `@test-driven-development`、`@verification-before-completion`、`@systematic-debugging`。  
- **代码原则：** DRY、YAGNI、最小可行增量；每个任务都先写失败测试，再做最小实现。  
- **提交原则：** 每个任务完成即提交（小步快跑）。  
- **破坏式重构边界：** 仅“深度研究”链路重构；KB Chat / General Chat / 评测链路不改语义。  
- **MCP 约束：** 研究模块不使用 `load_mcp_tools`，不提供 MCP 工具入口。

## 2. 风险与回滚策略

- **风险 1：** deepagents 流式事件结构与现有 SSE 适配不一致。  
  **缓解：** 在 `ResearchEventStore` 层做事件标准化映射，前端只消费统一 envelope。  
- **风险 2：** 中断恢复分支多，容易产生悬挂状态。  
  **缓解：** 状态机层强制 `interrupted -> resumed -> running` 迁移校验，失败写 `failure_summary` 工件。  
- **风险 3：** 破坏式删除旧代码时出现隐式引用。  
  **缓解：** 增加 `test_no_legacy_research_dependencies` 并在 CI 前执行。
